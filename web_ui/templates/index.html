三种研判模式

从「不执行」到「真执行」再到「让 Claude 真用一下」——三层证据强度递增，按需选择。

模式一 · 静态扫 + AI 研判（不执行 Skill）

用 17 条静态规则扫所有文件（找已知恶意模式），再把 SKILL.md 文本送 Claude（kuaipao.ai 中转）当安全审计员判 SAFE / SUSPICIOUS / MALICIOUS。报告里会显示 skill 的目录结构和每个文件内容预览。Skill 代码全程不执行——任何人上传任何文件都安全，零风险。最快、最便宜（约 ¥0.1/次）。

A · 上传文件扫描（推荐）
B · 强调 AI 研判（同上传，也走 Claude API）

模式二 · 动态执行（Docker 跑代码 + 看真行为）

在 VM 的 Docker 容器里直接 python script.py / bash script.sh 跑 skill 的所有可执行文件，strace 抓系统调用、tcpdump 抓网络包。不调 Claude API，零成本。最强证据——syscall 不会撒谎。或者你已经有以前录好的 strace.log 目录，直接喂进来重新打分。

A · 现场跑（连 VM 真执行）
B · 吃旧证据（离线打分）

模式三 · Docker 里跑 Claude（验证 Agent 抗诱导）

在 VM 的 Docker 容器里启动 Claude CLI，把 skill 装到 ~/.claude/skills/，给 Claude 一个用户 prompt 让它"读这个 skill 并使用"——然后 strace 录下 Claude 实际做了什么。回答的核心问题：真 Claude 会不会被这个 SKILL.md 骗着干坏事。这才是参考文献真正关心的攻击面。约 ¥1.2/次。
