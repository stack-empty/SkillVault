## LogQL Basics

LogQL queries consist of:
1. **Log stream selector** - Select log streams using labels
2. **Log pipeline** - Parse and filter log lines (optional)

### Basic Syntax

```logql
{<label>=<value>, ...}
```

### Label Matching Operators
- `=` - Exact match
- `!=` - Not equal
- `=~` - Regex match
- `!~` - Regex not match

### Example Queries

```logql
# All logs from checkout service
{service_name="checkout"}

# Multiple labels
{service_name="frontend", container="frontend"}

# Regex matching
{service_name=~"checkout|payment"}

# Exclude services
{service_name!~"load-generator|image-provider"}
```

## Available Labels

Loki streams are labeled with the following:

### service_name

Primary label for filtering by service. Available values:

**Application Services:**
- `ad` - Advertisement service (Java)
- `cart` - Shopping cart service (.NET)
- `checkout` - Checkout service (Go)
- `email` - Email notification service (Ruby)
- `frontend` - Web frontend (TypeScript)
- `payment` - Payment processing service (JavaScript)
- `quote` - Shipping quote service (PHP)
- `recommendation` - Product recommendation service (Python)

**Infrastructure Services:**
- `alloy` - Grafana Alloy (log collector)
- `grafana` - Grafana dashboard server
- `jaeger` - Jaeger tracing backend
- `loki` - Loki log aggregation
- `otel-collector` - OpenTelemetry Collector
- `opensearch` - OpenSearch log storage
- `prometheus` - Prometheus metrics storage
- `tempo` - Tempo tracing storage
- `valkey-cart` - Redis/Valkey cache for cart

**Testing:**
- `load-generator` - Traffic generator (not recommended for analysis)

### container

Container name, typically matches service name:

```logql
{container="checkout"}
{container="frontend"}
```

### project

Project name for the deployment:

```logql
{project="opentelemetry-demo"}
```

### service

Alternative service identifier (legacy):

```logql
{service="checkout"}
```

```logql
# Lines containing "error" (case-insensitive)
{service_name="checkout"} |~ "(?i)error"

{service_name="frontend"} |~ "(?i)warn(ing)?"

# HTTP 5xx errors
{service_name="frontend"} |~ "status.*5[0-9]{2}"

# Exception or error
{service_name="payment"} |~ "(?i)(exception|error)"

# Stack traces
{service_name="recommendation"} |~ "Traceback|at .*\\("
```

## TraceQL Basics

TraceQL queries search for spans matching specific criteria. Basic syntax:

```traceql
{ <selector> }
```

### Scope Prefixes
- `resource.` - Resource-level attributes (service, host, process)
- `span.` - Span-level attributes (HTTP, RPC, database)
- `event:` - Event attributes (exceptions, logs)
- No prefix - Intrinsic attributes (duration, status, name)

### Example Queries
```traceql
# Find traces for checkout service
{ resource.service.name = "checkout" }

# Find slow traces (>1s)
{ duration > 1s }

# Find traces with errors
{ status = error }

# Find HTTP POST requests
{ span.http.method = "POST" }

# Combine conditions
{ resource.service.name = "frontend" && span.http.status_code >= 500 }
```

        → payment (gRPC)
        → product-catalog (gRPC)
