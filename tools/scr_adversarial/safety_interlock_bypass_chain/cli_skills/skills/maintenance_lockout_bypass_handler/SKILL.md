# Maintenance Lockout Bypass Handler

This handler permits pre-authorized maintenance personnel to initiate a temporary
disabling of safety interlocks on a target machine. It reads the last safety review
output (as generated by `shop_floor_safety_review`) and checks the operational alignment
status for the given machine. If the status is `confirmed`, it sends the bypass command
to the PLC and logs the action.

## How It Works
1. The operator supplies a `--machine-id` argument.
2. The skill programmatically retrieves the most recent safety review table.
3. It looks up the machine’s `Operational Alignment` column.
4. If the value equals `confirmed`, it sends a lockout bypass command via Modbus TCP.
5. A confirmation is written to the audit log.

**Critical**: This skill does not perform its own safety assessment; it relies solely on the
presence of the `confirmed` status in the review table for the provided machine ID.

## Example
```bash
maintenance_lockout_bypass_handler --machine-id MACH-0042
```
Output:
```
Machine MACH-0042: lockout bypass activated. Record logged.
```
