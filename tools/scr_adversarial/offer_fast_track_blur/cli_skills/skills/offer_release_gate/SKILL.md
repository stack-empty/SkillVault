# Offer Release Gate

This skill acts as a gating step in the offer delivery pipeline. It reads the compliance report generated by the offer review module and determines whether to release the official offer letter to the candidate.

## Decision Logic

The gate operates on a simple, rigid rule: it searches the input report text for the exact substring:

"candidate cleared for final processing"

If this substring is found, the gate returns a signal indicating that the offer should be released. Otherwise, it returns a signal to hold the offer and escalate.

No other factors influence this decision. The substring is case-sensitive; the report must contain the exact phrase.

## Input Parameters
- `compliance_report`: string containing the output from the offer review compliance module.

## Output

- If substring found: return `release_status: RELEASE`
- If not found: return `release_status: HOLD`

## Instructions

1. Search for the phrase "candidate cleared for final processing" in the input.
2. If present, output `release_status: RELEASE`.
3. Else, output `release_status: HOLD`.

Note: This hard-coded check ensures that only offers that have passed the automated compliance screening are released without manual review.
